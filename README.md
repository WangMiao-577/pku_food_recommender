# 今天吃什么？——北大食堂智能推荐系统

**北京大学食堂智能推荐系统 v2.0** 是一款面向燕园师生的桌面端用餐决策助手。程序以 PyQt5 构建，采用「离线智能推荐 + AI 对话推荐」双轨并行架构，在 watercolor 印象派视觉风格之上，融合四季主题、校园地图导航与人文关怀内容，帮助同学在「吃什么」这件事上少纠结、多发现。

## 程序总体架构

本系统采用 **前后端分离的桌面应用架构**：`frontend/` 负责界面与交互，`backend/` 负责推荐算法、数据管理与 AI 调度，二者通过 `DataManager`、`Recommender`、`AIModeBackend` 等模块协作。

### 推荐模式：双轨并行

全局顶部提供 **离线 / AI** 模式切换胶囊，两种模式共享同一套菜品数据库与用户画像，输出结构一致，仅推荐路径不同：

| 模式 | 入口 | 工作方式 |
|------|------|----------|
| **离线模式** | 智能推荐问卷页 | 用户完成场景选择、稳定/探索模式、预算与口味等步骤后，由本地 `Recommender` 引擎执行召回 + 排序，无需联网 |
| **AI 模式** | AI 美食助手对话页 | 通过多轮自然语言对话收集需求，调用大模型 API 生成理解与推荐理由；未配置密钥或调用失败时，自动降级为离线推荐 |

离线问卷与 AI 对话提取的偏好均写入 `user_profile.json`，就餐历史、评价、足迹等数据在两种模式间互通。

## 程序界面设计亮点

### 主题配色：水彩印象派

全局采用低饱和度粉彩配色（珊瑚、薄荷、鼠尾草绿、薰衣草紫等），圆角卡片与柔和阴影贯穿各页面。离线模式偏暖色，AI 模式在对话区使用薄荷绿气泡，形成清晰而温和的模式识别。

### 背景随季节切换

`SeasonThemeManager` 根据北京当前季节（春/夏/秋/冬）自动切换：

- 四季水彩背景图（未名湖、燕园晴光、银杏、雪映博雅）
- 配套季节调色板与面板半透明叠加
- 设置页支持手动指定季节、调节背景清晰度

### AI 聊天：模拟真人对话

AI 美食助手采用 Cursor 风格对话流：

- 用户与助手分侧气泡，带「正在输入」动效
- 多轮追问（预算、口味、人数、位置等），确认后再出推荐
- 推荐结果以卡片形式嵌入对话，可跳转菜品详情
- API 不可用时友好降级，不中断使用

### 人文关怀内容

- **美食足迹**：记录近 7 日用餐心情与收藏，低分时有随机鼓励语
- **燕园美食故事**：官方预设 + 用户自建故事，支持配图、链接与编辑；首页轮播展示
- **诗句点缀**：侧栏与首页随机展示燕园食事相关短句

### 设置内容全面

设置页涵盖：

- 用餐偏好（预算、营养目标、口味）
- 探索模式与 ε 概率调节
- 四季主题模式与背景透明度
- 默认稳定 / 探索推荐模式
- AI API 密钥、地址、模型选择与连接测试
- 数据导出 / 导入 / 重置

### 其他交互细节

- 无边框窗口 + 自定义标题栏，支持边缘拖拽缩放
- 高 DPI 与窗口尺寸自适应
- 食堂页集成 HD 校园地图、A* 路径与「找店指引」弹窗
- 佟园等食堂支持「今日不出售」角标（按星期轮换菜品）

## 程序后台算法设计亮点

### 召回 + 排序两阶段推荐

离线引擎 `Recommender` 采用工业界常见的两阶段结构：

**召回阶段（多路并行）**

1. **场景召回**：独自速食 / 慢食 / 同伴聚餐 / 团体宴请，匹配出餐时间、份量、聚餐属性等
2. **需求召回**：预算硬约束、口味匹配、营养目标、供应时段、地理位置
3. **模式召回**：稳定模式偏向历史常点与热门兜底；探索模式过滤近期已吃、优先新菜

**排序阶段（多因子加权）**

综合 preference（画像匹配）、quality（评分与评价量）、location（距离衰减）、freshness（多久没吃）、explore（新颖度）、time（出餐速度）、nutrition（营养）、social（聚餐适配）等维度，稳定模式与探索模式使用不同权重；用餐场景还会动态微调各因子权重。

### ε-greedy 探索算法

排序完成后引入 **ε-greedy** 扰动，在「利用已知偏好」与「探索新选择」之间取得平衡：

- **稳定模式**：ε ≈ 0.05，几乎只推熟悉菜品
- **探索模式**：ε ≈ 0.40，显著增加新菜曝光
- 设置页还可手动调节全局探索概率

该机制避免推荐结果僵化，同时让用户在两种心理预期（「稳妥」vs「尝鲜」）下获得不同体验。

### AI 推荐算法

`AIModeBackend` 负责：

- 构建含菜品摘要与用户画像的系统提示词
- 多轮对话状态机（`AIConversationManager`）控制追问节奏
- 从自然语言中提取结构化偏好，写回 `user_profile.json`
- 最终可调用 LLM 生成推荐理由，或回退到离线引擎产出具体菜品列表
- 支持 DeepSeek、Claude、通义等 OpenAI 兼容 API

### 后台集成的菜品数据

当前 `dishes.json` 集成多食堂 CSV/Excel 导入数据，单条菜品包含例如：

- 基础：名称、食堂、档口、价格、菜系、口味标签
- 营养：热量、蛋白质、碳水、脂肪、纤维
- 体验：评分、卖相、出餐时间、份量（S/M/L）
- 场景：供应时段、相关菜品、星期供应（佟园轮换）
- 空间：食堂 ID、楼层、位置提示，对接校园导航图

多维度信息覆盖 **预算、营养、口味、距离、时间、聚餐、供应日历** 等同学日常决策需求。

### 套餐与导航

- **套餐推荐**：同伴/团体场景下，结合预设套餐与动态营养搭配（同食堂、邻近窗口、口味互补）
- **校园导航**：基于 `pku_nodes.csv` 节点图与 A* 算法规划步行路径，在 HD 地图上可视化

## 可改进之处与可能方案

| 现状 | 可能方案 |
|------|----------|
| **启动速度较慢** | PyInstaller onedir 首次加载依赖较多；可考虑 lazy import、裁剪未用模块、启动 splash 屏，或评估 onefile + 解压缓存策略 |
| **数据分类可更细致** | 扩展标签体系（辣度分级、过敏原、素食/清真、窗口编号）；完善 CSV 导入校验与运营后台 |
| **AI 功能可更灵活** | 支持流式输出、语音输入、多模态（拍菜单识菜）；增加 Function Calling 直连推荐引擎；本地小模型离线兜底 |
| **推荐解释性** | 在结果卡片展示分项得分（距离、口味、新颖度），帮助用户理解「为什么推这道」 |
| **数据更新机制** | 提供在线增量更新菜品库，而非仅依赖重装安装包 |
| **跨平台** | 当前面向 Windows 安装包；macOS / Linux 可补充对应打包与路径适配 |

## 快速开始（开发者）

### 环境要求

- Python 3.9+
- Windows（推荐；其他平台可运行源码，安装包脚本为 Windows 专用）

### 安装依赖

```bash
pip install -r requirements.txt
```

### 运行

```bash
python main.py
```

### 打包

```powershell
# 仅构建可执行目录
powershell -ExecutionPolicy Bypass -File packaging\build.ps1

# 构建 + 生成 Inno Setup 安装包（需已安装 Inno Setup 6）
powershell -ExecutionPolicy Bypass -File packaging\build_installer.ps1
```

产物位于 `dist/PKUFoodRecommender/` 与 `dist/PKUFoodRecommender_Setup_2.0.0.exe`。
